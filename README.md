# Голос по фактам

[Открыть каталог](https://cozycactus.github.io/golos-po-faktam/) ·
[Открыть досье № 43884](https://cozycactus.github.io/golos-po-faktam/votes/43884/)

Русскоязычный civic-tech прототип, который показывает заявления политиков рядом
с официальными текстами законов и поимёнными голосованиями Кнессета.

Главная страница — каталог проверенных досье. Первое досье посвящено голосованию
`43884` по поправке № 4 к Основному закону: Судебная власть. Оно отделяет
окончательное третье чтение от голосований по отдельным поправкам, показывает
все 70 зарегистрированных результатов и объясняет границы возможного вывода.

## Как устроены досье

- `data/votes/<id>.ts` хранит тексты, ссылки и проверенный снимок результатов;
- `data/votes/index.ts` регистрирует досье в каталоге;
- `/votes/<id>/` получает свежую поимённую запись из OData API Кнессета и при
  недоступности API использует снимок;
- статическая сборка GitHub Pages заранее создаёт каталог и физическую вложенную
  страницу каждого зарегистрированного досье.

## Источники

- `KNS_PlenumVote` — сведения о голосовании;
- `KNS_PlenumVoteResult` — поимённые результаты;
- официальный опубликованный текст закона в базе Кнессета.

При недоступности внешнего API сайт использует проверенный снимок тех же 70
строк. Отсутствие депутата в поимённой записи никогда не трактуется как голос
«против».

## Локальный запуск

```bash
npm install
npm run dev
```

Проверка перед публикацией:

```bash
npm run build
npm run lint
npm run typecheck:pages
npm test
npm run test:pages
npm audit
```

`npm run build` собирает серверную версию для Sites. `npm run build:pages`
создаёт полностью статическую версию в `dist-pages/` для GitHub Pages. При
публикации ветки `main` GitHub Actions автоматически проверяет и обновляет сайт.
